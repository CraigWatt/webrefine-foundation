# WebRefine Foundation Shape

This document defines what WebRefine is, what it is not, and how it should evolve.

## What WebRefine Is

WebRefine is a brand-aware starter foundation for building modern web apps and websites.

It combines:

- reusable design and layout primitives
- starter templates for chosen frameworks
- AI-readable guidance for consistent generation
- optional Three.js / React Three Fiber support for motion-rich interfaces

## What WebRefine Is Not

WebRefine is not:

- a giant all-purpose component library
- a framework that hides product decisions
- a template marketplace full of unrelated starter styles
- a 3D-only system

## Product Shape

The lean long-term shape is:

- one installable foundation package
- thin Next.js and Astro starters built on top of it
- documentation that acts as the style guide and AI contract

## How It Should Help A New App

When starting a new project, WebRefine should provide:

- a clear page shell
- a small set of layout primitives
- tokens for typography, spacing, color, and motion
- a reusable scene wrapper for optional R3F work
- enough examples to make extension obvious

## Core Rules

- structure comes before decoration
- brand comes through tokens, type, pacing, and motion
- components stay few, flexible, and composable
- framework wrappers stay thin
- canvas is an enhancement layer, not the default identity

## Build Priorities

The next practical work should focus on:

1. tokens
2. layout primitives
3. type scale and rhythm
4. starter wrappers for Next.js and Astro
5. richer but still restrained scene examples

## One-Sentence Summary

WebRefine is a reusable foundation for shipping branded web experiences quickly, with enough structure to be dependable and enough flexibility to avoid looking templated.
