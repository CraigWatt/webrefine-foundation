# WebRefine Design Meta

This document defines the visual and structural point of view for WebRefine starters.

## Core Position

WebRefine should feel like a brand-led starter system, not a generic component kit.

The foundation should help teams launch quickly without forcing every project into the same personality.

## Reference Direction

The target balance is:

- the restraint, clarity, and composure of premium service-led sites
- the confidence, distinctiveness, and art direction of strong design studios

That means starters should feel intentional and polished, but not over-designed or difficult to extend.

## Principles

### 1. Structure First

Pages should read clearly before decoration is added.

- strong hierarchy
- obvious section rhythm
- clear CTAs
- predictable layout primitives

### 2. Brand Over Boilerplate

The foundation should support distinctive brand expression without requiring a redesign of the underlying layout system.

- expressive typography
- controlled color systems
- intentional spacing
- motion that reinforces tone

### 3. Restraint Wins

Premium work usually feels edited.

- fewer components
- fewer competing colors
- fewer gimmicks
- better pacing

### 4. Motion With Purpose

Animation should guide attention, reveal hierarchy, or create atmosphere.

- avoid decorative motion for its own sake
- use stagger, parallax, transitions, and canvas motion sparingly
- let 3D scenes support the brand, not dominate it by default

### 5. Reusable Foundations, Specific Outcomes

The same starter primitives should be able to support multiple personalities.

- layouts should be reusable
- tokens should be adaptable
- scenes should be modular
- framework wrappers should stay thin

## What Starters Should Usually Include

- a clear page shell
- a small set of layout primitives
- one opinionated type scale
- one restrained color system
- one motion language
- optional React Three Fiber canvas support

## What Starters Should Avoid

- generic SaaS visuals by default
- oversized component inventories
- styling decisions that only work for one niche
- animation patterns that make content harder to parse
- canvas scenes that overpower mobile performance

## Practical Implications

For WebRefine, this means:

- start with sections and page rhythm before building lots of components
- treat typography and spacing as core primitives
- let brand choices happen through tokens and art direction
- use R3F as a supported enhancement layer, not the only identity of the system

## Meta Statement

WebRefine is a structured, brand-aware starter foundation for modern web apps and websites, with optional 3D canvas support and enough visual intent to avoid looking templated.
